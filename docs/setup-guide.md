# SOP: Creating and Deploying a Docusaurus Documentation Site on GitHub Pages

**Project Name:** DY_Documents  
**Repository:** DY_Documents  
**Author:** Deepti Yadav

---

# 1. Objective

Create a centralized documentation website for storing technical notes and guides related to:

- Commvault
- DevOps
- Kubernetes
- Docker
- CI/CD
- Linux
- GitHub Actions

The website will be hosted using **GitHub Pages** and built using **Docusaurus**.

---

# 2. What is Docusaurus?

Docusaurus is a static site generator developed by Meta (Facebook) that converts Markdown files into a professional documentation website.

## Benefits

- Easy to maintain
- Markdown-based
- Automatic sidebar generation
- Search support
- Dark mode
- GitHub Pages deployment
- Version control integration

# 6. Understanding Each Folder

## docs/

Contains all documentation pages written in Markdown.

Example:

```text
docs
│
├── intro.md
├── Commvault
│     └── Introduction.md
└── DevOps
      ├── Introduction.md
      └── setup-guide.md
```

---

## src/

Contains custom CSS and React components.

For simple documentation:

```text
src
└── css
      └── custom.css
```

---

## static/

Stores static files.

Example:

```text
static
├── img
│     ├── logo.svg
│     ├── favicon.ico
│     └── docusaurus-social-card.jpg
└── .nojekyll
```

---

## .github/workflows/

Contains GitHub Actions pipelines.

```text
.github
└── workflows
      └── deploy-pages.yaml
```

Used for automatic deployment.

# 7. sidebars.js

## Purpose

Controls navigation menu.

## File

```javascript
const sidebars = {
  tutorialSidebar: [
    {
      type: 'autogenerated',
      dirName: '.',
    },
  ],
};

module.exports = sidebars;
```

### Why?

Automatically generates the sidebar based on the folder structure.

---

# 8. docusaurus.config.ts

## Purpose

Main configuration file.

### Website URL

```ts
url: 'https://deeptiyadavsd1111.github.io',
baseUrl: '/DY_Documents/',
```

### Home Page

```ts
docs: {
  routeBasePath: '/',
  sidebarPath: './sidebars.js',
},
```

Meaning:

```text
https://deeptiyadavsd1111.github.io/DY_Documents/
```

opens the documentation directly.

### Broken Links

```ts
onBrokenLinks: 'throw',

markdown: {
  hooks: {
    onBrokenMarkdownLinks: 'warn',
  },
},
```

  border-radius: 8px;
